Why is this an issue?

Declaring types (structs, enums, unions, typedefs) and variables in the same statement reduces code clarity and maintainability. Keeping these declarations separate makes code easier to read and understand.

Mixing type and variable declarations:

  • Reduces code clarity and readability
  • Makes it harder to find type definitions
  • Complicates code navigation and searching
  • Violates separation of concerns principle
  • Can lead to confusion about scope and visibility

Example:

    // Noncompliant - type and variable in same statement
    typedef struct {
        int x;
        int y;
    } Point, myPoint;  // Type definition and variable declaration mixed
    
    // Noncompliant - enum type and variable together
    enum Color { RED, GREEN, BLUE } myColor;
    
    // Noncompliant - struct definition and variable in one statement
    struct Employee {
        char name[50];
        int id;
    } emp1, emp2;
    
    // Compliant - type definition separate from variable declarations
    typedef struct {
        int x;
        int y;
    } Point;
    
    Point myPoint;
    
    // Compliant - enum type separate from variable
    enum Color { RED, GREEN, BLUE };
    
    enum Color myColor = RED;
    
    // Compliant - struct definition separate from variables
    struct Employee {
        char name[50];
        int id;
    };
    
    struct Employee emp1;
    struct Employee emp2;
    
    // Compliant - multiple declarations of same type
    int a, b, c;  // OK - same type, all variables
    struct Point p1, p2;  // OK - same type, all variables

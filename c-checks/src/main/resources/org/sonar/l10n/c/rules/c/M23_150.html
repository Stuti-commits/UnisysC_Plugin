Why is this an issue?

A function declared with a non-void return type must return a value on all execution paths. If a function doesn't return a value on some path, the caller receives an undefined/uninitialized value, leading to unpredictable behavior.

This violates the C standard and can result in:

  • Undefined behavior
  • Data corruption
  • Logic errors that are difficult to debug
  • Compiler warnings or errors

All code paths in a non-void function must explicitly return a value, including:

  • Normal execution paths
  • Error handling branches
  • Loop exit paths
  • Default cases in switch statements

Example:

    // Noncompliant - missing return on some paths
    int getValue(int flag) {
        if (flag > 0) {
            return 10;
        }
        // Missing return statement when flag <= 0
    }
    
    // Noncompliant - switch without default case
    int getStatus(int code) {
        switch (code) {
            case 0:
                return 0;
            case 1:
                return 1;
            // Missing return for unknown codes
        }
    }
    
    // Compliant - all paths return a value
    int getValue(int flag) {
        if (flag > 0) {
            return 10;
        }
        return 0;  // Return on all paths
    }
    
    // Compliant - default case handles all cases
    int getStatus(int code) {
        switch (code) {
            case 0:
                return 0;
            case 1:
                return 1;
            default:
                return -1;  // Handles all other cases
        }
    }

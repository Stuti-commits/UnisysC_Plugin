Why is this an issue?

Returning a pointer or reference to a local variable with automatic storage duration is undefined behavior. When the function returns, the local variable goes out of scope and its memory is deallocated, leaving the caller with a dangling pointer or reference.

Any attempt to dereference such a pointer or reference results in accessing freed memory, which can lead to crashes, data corruption, and security vulnerabilities.

Example:

    int* getLocalPointer() {
        int local = 42;
        return &local;         // Noncompliant - returning pointer to local variable
    }
    
    int* getStaticPointer() {
        static int staticVar = 42;
        return &staticVar;     // OK - static variable persists
    }
    
    int* getHeapPointer() {
        int *p = malloc(sizeof(int));
        *p = 42;
        return p;              // OK - dynamically allocated memory
    }
    
    void caller() {
        int *ptr = getLocalPointer();
        printf("%d\n", *ptr);  // Undefined behavior - dangling pointer
    }

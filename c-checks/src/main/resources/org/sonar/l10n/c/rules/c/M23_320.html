Why is this an issue?

Escape sequences in string literals must be clearly terminated to avoid ambiguity and unintended behavior. If an octal or hexadecimal escape sequence is followed by additional characters, it may be misinterpreted.

Noncompliant code example

const char *s1 = "\1234";   // Noncompliant - \123 is not terminated
const char *s2 = "\x41g";   // Noncompliant - \x41 is not terminated

Compliant solution

const char *s3 = "\123";        // OK - properly terminated
const char *s4 = "\x41\x67";    // OK - terminated by another escape

Limitations

Unicode escape sequences (\u and \U) are not checked due to limitations in the current grammar support.
